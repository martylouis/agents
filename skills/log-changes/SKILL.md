# Log Changes

Capture a reasoning-first log entry for the current session's work.

## Quick Start

```
/log-changes
```

Creates a log entry with a summary, rationale, and files touched.

## Workflow

### Phase 1: Preflight

Check that we're in a git repo with changes:

```bash
git rev-parse --is-inside-work-tree 2>/dev/null || echo "NOT_GIT"
git status --porcelain
```

- If not a git repo → display: "Not a git repository."
- If no changes → display: "No changes to log. Make some changes first."

### Phase 2: Gather Context

```bash
# Timestamps
date +"%Y-%m-%d"           # → DATE
date +"%H%M"               # → TIME
date +"%Y-%m-%d %H:%M"     # → DATE_DISPLAY

# Branch
git branch --show-current

# Changed files (all — staged, unstaged, untracked)
git status --porcelain
```

### Phase 3: Infer from Conversation

**Category** — Infer from conversation context and changes:

| Category | When to use |
| -------- | ----------- |
| `feat` | New features, new functionality |
| `fix` | Bug fixes, error corrections |
| `chore` | Maintenance, cleanup, refactoring, dependencies |
| `docs` | Documentation changes |
| `perf` | Performance improvements |

Default to `feat` if unclear.

**Title** — Derive from conversation:

1. Extract the main topic or feature
2. Convert to slug: lowercase, hyphens, no special characters
3. Keep concise (3-5 words max)

**Summary** — 2-3 sentences of what was accomplished.

**Rationale** — Synthesize from the conversation:
- Why were these changes made?
- What decisions were made and why?
- What alternatives were considered?
- Any constraints that shaped the approach?

**If the conversation doesn't make the reasoning clear**, ask 1-2 targeted questions. Examples:
- "What motivated this change?"
- "Were there alternatives you considered?"

Only ask if you genuinely can't infer the rationale. Most conversations contain enough context.

### Phase 4: Ask Output Location

Ask the user where to save the log:

```
Where should the log be saved?
1. `./docs/logs/` (project documentation)
2. `./.claude/docs/logs/ (local claude workspace)
```

Use their choice for the output path. If they've answered before in this project, use the same location without asking again.

### Phase 5: Write the Log

**Output path:** `{location}/{DATE}-{TIME}-{category}-{title}.md`

Create the directory if needed: `mkdir -p {location}`

**Template:**

```markdown
# {Title (human readable)}

| Field | Value |
| ----- | ----- |
| Date | {DATE_DISPLAY} |
| Branch | {branch} |
| Category | {category} |

## Summary

{2-3 sentence overview of what was accomplished}

## Rationale

{Why these changes were made. Capture the reasoning, decisions, tradeoffs, and constraints from the conversation. Write in clear prose, not bullet points — unless bullets genuinely fit better.}

## Files

- `path/to/file1.ext`
- `path/to/file2.ext`

---

_Generated by log-changes_
```

### Phase 6: Confirmation

Display:

```
Logged: {full_absolute_path}
Category: {category}
Title: {title}
```

Use `pwd` to build the full absolute path so it's clickable in terminal.

## Edge Cases

| Situation | Behavior |
| --------- | -------- |
| No changes | Error: "No changes to log. Make some changes first." |
| Not a git repo | Error: "Not a git repository." |
| Thin conversation context | Ask 1-2 targeted questions about rationale |
| No conversation context | Use branch name and file changes to infer title; ask about rationale |
| Ambiguous category | Default to `feat` |
| Output directory missing | Create with `mkdir -p` |
